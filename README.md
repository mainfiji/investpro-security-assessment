# Оценка защищённости и реализация мер безопасности в компании


## О проекте

Компания «ИнвестПро» проводит еженедельные инвестиционные сессии, от которых зависят финансовые результаты на фондовом рынке.  
В рамках этого проекта я:

1. **Проанализировал текущую защищённость** инфраструктуры компании, выявил уязвимости и угрозы.
2. **Доработал модель угроз и оценку рисков** с учётом новых бизнес-требований (публикация отчётности на сайте, удалённая работа и т.д.).
3. **Реализовал предложенные меры защиты** в виртуальном стенде:
   - Харденинг ОС (минимальные привилегии, парольная политика).
   - Защита веб-сервера (WAF, перенаправление трафика, запрет доступа к системным файлам).
   - Защита электронной почты (антиспам/антифишинг, блокировка запароленных архивов).
   - Защита файловых ресурсов (Samba с разграничением доступа по группам).
   - Установка антивируса (ClamAV) и его интеграция с Samba (опционально).
   - Фильтрация трафика (nftables) на основе принципа нулевого доверия.

**Результат:** Инфраструктура приведена к безопасному состоянию, все ключевые сервисы работают, риски снижены, модель угроз актуализирована.

## Выполненные задачи

### Часть 1. Анализ защищённости и моделирование угроз

| Задача | Результат |
|--------|-----------|
| Сбор информации (интервью, изменения) | Выявлены новые активы: веб-сервер, публикация отчётности, удалённый доступ через RDP |
| Изучение сети из интернета | Обнаружены открытые порты: 22, 25, 80, 443, 143, 993, 3389, 4307 и др. |
| Изучение сети изнутри | Составлена таблица хостов, их ОС, сервисов и найденных уязвимостей |
| Анализ эндпоинтов | Выявлены проблемы: отсутствие 2FA, слабая аутентификация, ручное управление правами, устаревшее ПО |
| Проверка защиты почты | Найден пользователь, открывающий подозрительные вложения (boss) |
| Доработка модели угроз | Добавлены новые недопустимые события, угрозы, уязвимости; предложены меры защиты с приоритетами |
| Подготовка новой схемы сети | Разработана схема с выделением DMZ, сегмента управления, WAF, VPN-шлюза |

### Часть 2. Реализация мер защиты

| Задача | Результат |
|--------|-----------|
| Харденинг ОС | Настроены минимальные привилегии для пользователей, парольная политика, изменён пароль boss |
| Защита веб-сервера | Трафик перенаправлен через WAF (BunkerWeb), настроены правила против перебора директорий, запрещён доступ к /log/ |
| Защита электронной почты | Настроен rspamd: блокировка писем с запароленными архивами, включена проверка архивов |
| Защита файловых ресурсов | Настроена Samba: созданы группы (boss, managers, analysts, admins), выданы права на каталоги strategy, reports, clients, it; анонимный доступ запрещён |
| Антивирусная защита | Установлен ClamAV на все хосты, обновлены базы, протестирован детект EICAR (опционально: интеграция с Samba) |
| Фильтрация трафика | Разработаны и применены правила nftables, разрешены только необходимые потоки (по таблице), остальное запрещено |
| Проверка доступности | Все сервисы работают, из интернета доступны только WAF (80/443) и почта (25/587/993), SSH/RDP только через VPN (планируется) |

## Мои выводы

Проект позволил мне:
- На практике применить **моделирование угроз** и оценку рисков для реального бизнес-процесса.
- Провести **анализ защищённости** инфраструктуры с использованием nmap, сканирования портов, проверки конфигураций.
- Реализовать **комплексные меры защиты**:
  - Сетевую фильтрацию на уровне nftables.
  - Харденинг ОС и прикладных сервисов.
  - Установку и настройку WAF, антивируса, почтового фильтра.
  - Разграничение доступа на основе ролей (RBAC) для файлового сервера.
- Продемонстрировать понимание **принципа наименьших привилегий** и **нулевого доверия**.
