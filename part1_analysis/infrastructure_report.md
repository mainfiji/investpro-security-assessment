# Отчёт об исследовании инфраструктуры компании «ИнвестПро»

## 1. Сбор информации

### Беседа с директором по инвестициям

- Процесс проведения инвест-сессий не изменился, но теперь компания публикует отчётность на официальном сайте.
- Своевременность и полнота данных проверяется государственным регулятором, поэтому критически важно, чтобы данные были доступны и не искажены.

### Беседа с техническим директором

- Запущен собственный веб-сервер, так как генеральный директор не доверяет облачным сервисам.
- Планируется развернуть WAF, но сервер для него ещё не приехал.
- Устранены уязвимости в ПО: обновлены TrueConf, роутер Cisco.
- Ограничен доступ в серверную, наведён порядок с носителями, запрещены личные сетевые устройства.
- В перспективе — отказ от облачных хранилищ, возможно развертывание защищённого файлового хранилища для удалённых сотрудников.

### Беседа с главным администратором

- Веб-сервер на Nginx, WAF планируется, но сервер не приехал.
- Обновлены TrueConf и Cisco IOS.
- Пользователи переведены на Ubuntu 22.04.
- Доступ из интернета: проброс портов для почты, SSH, TrueConf, веб-сайт, RDP.
- Почта теперь на Dovecot, файловый сервер — Samba.
- Права доступа к файлам настраиваются вручную («по звонку»).
- Серверная ограничена, убран лишний роутер, выделены деньги на резервный канал.

## 2. Анализ защищённости инфраструктуры

### Проблемы безопасности архитектуры сети

1. **Прямой проброс портов RDP и SSH в интернет** — высокий риск несанкционированного доступа.
2. **Отсутствие WAF** для защиты веб-сервера.
3. **Недостаточная защита почтового сервера** (нет антиспама, антифишинга).
4. **Ручная настройка прав доступа к файловому серверу** — риск ошибок и нарушения принципа минимальных привилегий.
5. **Использование устаревших версий ПО** на некоторых системах (например, TrueConf до обновления).

### Комментарии по защите

- **WAF** обязателен для защиты веб-сервера от SQLi, XSS, DDoS.
- **VPN** вместо прямого проброса RDP/SSH снизит риск внешних атак.
- **Антиспам/антифишинг** для почтового сервера (rspamd уже установлен, требуется настройка).
- **Автоматизация управления правами** на файловом сервере (интеграция с LDAP/AD).
- **Сегментация серверной сети**: выделить DMZ для веб-сервера, отдельные VLAN для почты, ВКС, файлового хранилища.
- **Двухфакторная аутентификация** для всех административных доступов.
- **Регулярное обновление** всех систем и аудит безопасности.
- **Резервный интернет-канал** повысит отказоустойчивость.
- **Запретить прямой проброс RDP и SSH** в интернет, оставив только VPN.
- **Внедрить EDR/антивирус** на все рабочие станции и серверы.
