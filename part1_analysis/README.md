# Часть 1. Анализ защищённости и моделирование угроз

## Выполненные задачи

1. **Сбор информации** – интервью с руководством и администрацией.
2. **Анализ инфраструктуры** – изучение сети, открытых портов, эндпоинтов.
3. **Анализ эндпоинтов** – проверка версий ОС, прикладных сервисов, уязвимостей.
4. **Проверка защиты почты** – тест на фишинг, выявление пользователя boss.
5. **Доработка модели угроз** – актуализация недопустимых событий, уязвимостей, угроз.
6. **Формулирование мер защиты** – предложения по устранению уязвимостей, приоритеты.

## Ключевые выводы

- Исходная сеть имела ряд критических уязвимостей: прямой проброс портов, отсутствие WAF, слабые пароли, отсутствие сегментации.
- Модель угроз была дополнена новыми угрозами (DDoS, подмена контента, модификация данных).
- Предложены меры защиты с приоритетами: внедрение WAF, VPN, 2FA, сегментация, антиспам, антивирус.
